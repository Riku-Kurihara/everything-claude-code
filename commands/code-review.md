# コードレビュー

コミット前の変更に対する包括的なセキュリティと品質のレビュー:

1. 変更されたファイルを取得: git diff --name-only HEAD

2. 変更されたファイルごとに確認:

**セキュリティの問題 (重大):**
- ハードコードされたクレデンシャル、APIキー、トークン
- SQLインジェクション脆弱性
- XSS脆弱性
- 入力検証の欠落
- 安全でない依存性
- パストラバーサルのリスク

**コード品質 (高):**
- 50行を超える関数
- 800行を超えるファイル
- ネストの深さが4レベル以上
- エラーハンドリングの欠落
- console.log文
- TODO/FIXMEコメント
- パブリックAPI用のJSDocの欠落

**ベストプラクティス (中):**
- ミューテーションパターン (代わりにイミュータブルを使用)
- コード/コメント内の絵文字使用
- 新しいコードのテストの欠落
- アクセシビリティの問題 (a11y)

3. 以下を含むレポートを生成:
   - 重要度: 重大, 高, 中, 低
   - ファイルの場所と行番号
   - 問題の説明
   - 修正の提案

4. 重大または高の問題が見つかった場合はコミットをブロック

セキュリティ脆弱性があるコードは承認しないでください!
