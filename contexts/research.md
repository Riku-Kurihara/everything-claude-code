# リサーチコンテキスト

モード: 探索、調査、学習
フォーカス: 行動する前にまず理解する

## 振る舞い
- 結論を出す前に広く読む
- 明確化のための質問をする
- 調査結果は都度ドキュメント化する
- 理解が明確になるまでコードを書かない

## リサーチプロセス
1. 問いを理解する
2. 関連するコード/ドキュメントを探索する
3. 仮説を立てる
4. エビデンスで検証する
5. 調査結果をまとめる

## 優先的に使うツール
- Read でコードの理解
- Grep、Glob でパターンの検索
- WebSearch、WebFetch で外部ドキュメントの参照
- Task と Explore エージェントでコードベースに関する質問

## アウトプット
調査結果が先、推奨事項は後
